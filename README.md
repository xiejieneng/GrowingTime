# GrowingTime 成长时光微信小程序

一个用于记录婴儿成长历程的原生微信小程序 Demo，包含图片高质量压缩、按拍摄信息自动整理、成长相册和 AI 短视频生成入口。

## 功能

- 图片整理：从手机相册或相机选择原图，读取 JPEG EXIF 中的拍摄日期和 GPS 信息。
- 高清压缩：使用微信原生压缩接口并行处理，默认最长边 4096px、质量 92%，并保存为小程序本地持久文件。
- 体积保护：展示图编码后若不小于原图，会自动放弃该结果并直接保留原图，避免展示图比原图更大。
- 自动归档：按照片 EXIF 拍摄月份生成成长相册；无法读取拍摄日期时归入“未知时间”。
- 拍摄时间：优先读取标准 EXIF 子目录中的原始拍摄/创建时间；无法读取时显示“未知日期”，不会再误用上传日期。
- AI 视频：选择一张已整理照片，通过可灵图生视频 API 生成具有真实动作和镜头变化的 5 秒或 10 秒视频。
- 在线播放：成品支持云存储 `videoFileId` 或 HTTPS `videoUrl`，小程序会解析临时地址并在视频页内播放。
- AI 成品归档：生成完成后，云函数会优先把 MP4 转存微信云存储，避免第三方临时链接失效。
- 视频记录管理：进入管理模式后可多选或全选排队、失败及已完成记录；云存储成品与封面会同步清理。
- 本次处理管理：整理页支持多选或全选移除本次处理记录，不影响已经保存到成长相册的照片。
- 照片管理：相册支持多选和全选删除，同时清理本地持久文件、云端展示图/原图及照片索引。
- 省钱存储：默认只上传展示图和索引；开启高清备份后，仅 Wi-Fi 下额外上传原图。
- 账号体系：未登录时全部数据只存本机；微信身份注册登录后可恢复云端照片并同步本地历史数据。

## 使用方式

1. 用微信开发者工具导入当前目录。
2. 开通云开发后上传并部署 `cloudfunctions/account` 和 `cloudfunctions/createVideo`。
3. 在 `createVideo` 云函数配置中添加环境变量 `KLING_ACCESS_KEY` 和 `KLING_SECRET_KEY`。
4. 可选配置 `KLING_VIDEO_MODEL`；未配置时使用 `kling-v1-6`。
5. 创建云数据库集合 `users` 和 `photos`，分别用于账号与照片元数据索引。

AI 视频密钥只能配置在云函数环境变量中，不要写入小程序代码、Git 仓库或前端本地存储。未登录或未配置密钥时，小程序会明确提示，不再生成本地幻灯片模拟记录。

项目已在 `project.config.json` 中绑定正式小程序 AppID。代码仍保留测试号兼容逻辑：如果改成 `touristappid`，会自动禁用云开发并将照片保存在本机。

视频服务创建或查询完成后，返回以下任一组字段即可触发在线播放：

```js
{
  status: "completed",
  statusText: "已完成",
  videoFileId: "cloud://环境/成品视频.mp4",
  coverFileId: "cloud://环境/视频封面.jpg"
}
```

也可返回 `videoUrl`、`coverUrl` HTTPS 地址。使用外部地址时，需要在微信公众平台配置对应的服务器域名；优先推荐把成品上传云存储并返回 `videoFileId`。

## 说明

- 小程序端只能读取图片内已有的 EXIF GPS 坐标；要显示城市、医院、公园等地点名，需要接入腾讯地图或其他逆地理编码服务。
- “无损压缩”在移动端通常取决于原图格式和编码器能力。当前实现采用高质量 JPG 重编码，尽量保留尺寸和视觉清晰度，同时降低体积。
- 视频生成服务会优先复用已经上传到云存储的照片 fileID，并在任务完成后转存成品。
